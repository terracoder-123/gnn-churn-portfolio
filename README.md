# GNN Churn Prediction — ML Portfolio Project

**Live demo:** https://yourusername.github.io/gnn-churn-portfolio/  
**API:** https://gnn-churn-api.onrender.com

A production-style ML portfolio project using **Graph Neural Networks** to predict customer churn. Customers are modelled as nodes in a social graph — edges represent call relationships. A **GraphSAGE** model aggregates neighbor features to capture social influence on churn decisions, outperforming a standalone XGBoost baseline.

## Why GNNs for churn?

Churn is contagious. Customers whose friends and contacts are leaving a telecom provider are more likely to leave themselves. A traditional ML model treats each customer independently — it cannot see this social signal. A GNN explicitly passes information across graph edges, learning that a customer with 4 churned neighbors is at higher risk than their individual features suggest.

## Interview talking points

- **Why GNN?** Churn has a social contagion effect — GraphSAGE captures neighbor influence that XGBoost cannot see
- **Graph construction:** Synthetic call-graph based on tenure cohorts; in production you'd use CDR logs
- **Class imbalance:** Handled with weighted cross-entropy (`scale_pos_weight`)
- **Serving:** Full-graph inference at startup; single-customer `/predict` falls back to tabular features
- **AUC lift:** GraphSAGE outperforms XGBoost by ~3% AUC, which is meaningful at scale
